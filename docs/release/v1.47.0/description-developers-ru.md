# Helyx v1.47.0 — Техническое описание для разработчиков

## Проблема

Claude Code — мощный локальный ассистент разработки, но он привязан к терминалу. Отошли от машины, закрыли крышку ноутбука, хотите быстро проверить статус — доступа нет. Нет встроенного способа получить уведомление о завершении длинной задачи, подтвердить запрос на разрешение в середине сессии или отправить следующее сообщение без клавиатуры.

Helyx решает это, делая Telegram панелью управления для одной или нескольких сессий Claude Code на любой доступной машине.

---

## Архитектура

Helyx состоит из двух частей, общающихся через PostgreSQL message bus.

### Хостовая часть: `channel.ts` (stdio MCP)

Bun-процесс на хосте регистрируется как MCP-сервер через stdio. Claude Code подключается к нему при старте через стандартный MCP stdio-транспорт. Этот канальный процесс:

- Инжектирует входящие Telegram-сообщения (текст, транскрибированный голос, описания фото) в контекст Claude Code — как если бы пользователь их ввёл.
- Захватывает вывод Claude Code (вызовы инструментов, ответы, запросы разрешений, статусы) и пересылает их в бот-процесс через PostgreSQL NOTIFY.
- Следит за tmux-панелью для живых обновлений «Думаю… (N:NN)».

Хостовый процесс запускается напрямую под Bun — он **не контейнеризирован**, потому что должен разделять файловую систему и пространство процессов с Claude Code.

### Docker-часть: HTTP MCP сервер + бот + дашборд

Docker Compose-стек включает:

- **Bot-процесс** (`bot.ts`, grammY): вся Telegram-интеграция — маршрутизация сообщений, рендеринг инлайн-кнопок, транскрипция голоса, обслуживание Mini App.
- **HTTP MCP сервер**: принимает результаты вызовов инструментов от Claude Code (чтение/запись памяти, предложения навыков, исходы разрешений) по HTTP. Это «MCP-сервер» с точки зрения Claude Code для всех non-stdio инструментов.
- **Веб-дашборд** (React + Vite, порт 3847): admin SPA — сессии, браузер памяти, логи разрешений, статистика API, git-браузер.
- **PostgreSQL 16 + pgvector**: центральная шина сообщений + постоянное хранилище. Векторы создаются через Ollama `nomic-embed-text` на хосте.

### Поток данных (типичное сообщение)

```
Пользователь отправляет сообщение в Telegram
  → bot.ts получает через grammY webhook/polling
  → вставляет в очередь PostgreSQL (NOTIFY)
  → channel.ts забирает через LISTEN
  → пишет в stdin Claude Code (MCP-сообщение)
  → Claude Code обрабатывает, создаёт ответ/вызов инструмента
  → вызов инструмента попадает на HTTP MCP сервер в Docker
  → HTTP-сервер вставляет результат в PostgreSQL
  → bot.ts доставляет ответ в Telegram-тред
```

## Быстрый старт (4 шага)

**Требования:** Docker + Docker Compose, Bun ≥ 1.1, установленный Claude Code, Anthropic API-ключ, Telegram Bot Token (от @BotFather), Ollama с загруженной моделью `nomic-embed-text`.

1. **Клонировать и настроить**
   ```bash
   git clone https://github.com/MrCipherSmith/helyx
   cd helyx
   cp .env.example .env
   # Отредактировать .env: TELEGRAM_BOT_TOKEN, ANTHROPIC_API_KEY, POSTGRES_*, OLLAMA_HOST
   ```

2. **Поднять стек**
   ```bash
   docker compose up -d
   ```

3. **Добавить MCP-сервер в Claude Code**
   ```json
   // .claude/settings.json вашего проекта
   {
     "mcpServers": {
       "helyx": {
         "command": "bun",
         "args": ["/path/to/helyx/channel.ts"]
       }
     }
   }
   ```

4. **Запустить Claude Code с MCP** — `claude` в вашем проекте. Бот появится в Telegram.

## Ключевые подсистемы

### Forum Supergroup Mode

Когда `TELEGRAM_FORUM_CHAT_ID` указан, каждый проект в таблице `projects` получает отдельный форум-топик. Маршрутизация сообщений, разрешений и статусов происходит по `forum_topic_id` — пользователю не нужно переключать сессии вручную. Форум-топик создаётся через Bot API при добавлении проекта через `/project_add`.

### Гейтинг разрешений

Вызовы Bash и Edit по умолчанию требуют подтверждения. Инструмент MCP `request_permission` вставляет запись в таблицу `permission_requests`; бот отправляет сообщение с кнопками Approve/Deny. Авто-апрув паттерны хранятся в таблице `auto_approve_rules` и проверяются до показа кнопок.

### Skills Toolkit

Когда Claude Code завершает значимую задачу, MCP-инструмент `propose_skill` вставляет предложение в `skill_proposals`. Бот отправляет его на проверку через инлайн-кнопку. Подтверждённые навыки хранятся в `agent_skills` и загружаются в контекст Claude через инструмент `list_agent_skills`. Еженедельный куратор-агент анализирует использование и архивирует устаревшие.

### Семантическая память

`remember()` создаёт запись в `memories` с pgvector-эмбеддингом через Ollama. `recall()` выполняет косинусный поиск ближайших соседей. `rememberSmart()` сначала ищет похожие записи через embedding-поиск и передаёт их LLM для проверки дублирования перед сохранением. При недоступности Ollama используется `embedSafe()` — возвращает null вместо исключения, recall откатывается к поиску по актуальности.

### Автосуммаризация

Supervisor-процесс следит за сессиями через `active_status_messages`. Когда heartbeat устаревает (Claude остановился), он запускает summarization-задачу, которая записывает краткое содержание в `memories` как `type='summary'`. Это содержание загружается следующей сессией через `/switch`.
