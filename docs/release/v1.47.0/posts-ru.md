## Habr — Заголовок статьи + Лид (первые ~300 слов)

**Заголовок:** Helyx: самохостинг Telegram-моста для Claude Code с MCP-транспортом, pgvector-памятью и форум-маршрутизацией

**Лид (начало статьи):**

Claude Code — серьёзный инструмент для разработки. Но у него есть одно принципиальное ограничение: он живёт в терминале. Отошёл от ноутбука — потерял контроль над сессией. Нет нативного способа получить уведомление о завершении задачи, подтвердить запрос разрешения или скинуть дополнительный контекст, не открывая SSH.

Я решил это радикально: сделал Telegram управляющей плоскостью для Claude Code. Результат — Helyx, открытый инструмент на Bun + TypeScript, который сейчас вышел в версии 1.47.0.

### Как это работает (кратко)

Архитектура состоит из двух рантаймов, общающихся через PostgreSQL.

**Хост**: процесс `channel.ts` на Bun регистрируется как MCP-сервер через stdio. Claude Code подключается к нему при старте. Канал принимает сообщения из Telegram и пишет их в stdin Claude Code; перехватывает ответы и передаёт обратно через PostgreSQL NOTIFY.

**Docker**: Compose-стек содержит бот-процесс (grammY), HTTP MCP-сервер (принимает вызовы инструментов от Claude Code), PostgreSQL 16 + pgvector, веб-дашборд (React + Vite, порт 3847).

Поток данных выглядит так:

```
Telegram → bot.ts → PostgreSQL NOTIFY → channel.ts → Claude Code stdin
Claude Code → HTTP MCP-сервер → PostgreSQL → bot.ts → Telegram
```

### Что добавилось в v1.47.0

**Forum Supergroup mode.** Каждый проект — отдельный топик в Telegram-форуме. Статус, ответы и запросы разрешений попадают в нужный топик автоматически. Никаких `/switch`-команд.

**Skills Toolkit.** После нетривиальных задач Claude предлагает сохранить паттерн как «навык» — имя, описание, тело промпта. Вы апрувите одной кнопкой в Telegram. Еженедельный куратор анализирует использование навыков, архивирует устаревшие, постит закреп-сводку.

**Гейтинг разрешений.** Деструктивные вызовы инструментов (редактирование файлов, shell-команды) появляются как инлайн-кнопки «Разрешить / Отклонить» в Telegram. Claude Code ждёт ответа.

**Семантическая память.** PostgreSQL + pgvector, дедупликация через LLM (`rememberSmart`), автосуммаризация на простое. Контекст накапливается между сессиями.

**Голос.** Транскрипция через Groq Whisper (или локальный фоллбэк), TTS-ответы через Piper, Kokoro, Яндекс или Groq.

Честно о сложности: нужны Docker, Bun, tmux, Ollama (для эмбеддингов `nomic-embed-text`) и две MCP-регистрации в конфиге Claude Code. Не однокнопочная установка, но управляемо, если вы уже держите self-hosted стек.

Полная документация и исходники: https://github.com/MrCipherSmith/helyx

## Telegram — Пост для dev-канала

Выложил v1.47.0 Helyx — свой open-source инструмент для управления Claude Code из Telegram 🤖

Коротко что это: self-hosted Telegram-бот, который делает Claude Code доступным с телефона. Пишешь в Telegram — сообщение уходит в Claude Code. Claude работает с кодом, отвечает в тот же тред.

**Что появилось в этой версии:**

📁 **Форум-режим** — каждый проект живёт в отдельном топике Telegram-форума, маршрутизация автоматическая

🧠 **Skills Toolkit** — Claude предлагает сохранить повторяемые паттерны как «навыки», апрувишь одной кнопкой

🔐 **Кнопки разрешений** — вместо ввода в терминале: «Claude хочет отредактировать auth/login.ts — Разрешить / Отклонить»

🔎 **Семантическая память** — pgvector + Ollama, контекст накапливается между сессиями

🎙 **Голос** — транскрипция входящих + TTS для ответов

Стек: Bun, TypeScript, grammY, PostgreSQL 16 + pgvector, Docker, React+Vite.

Честно: нужны Docker, tmux и Ollama. Не для тех, кто ищет одну кнопку «установить». Для тех, кто готов поднять self-hosted стек и хочет нормальный remote-доступ к AI-сессиям.

MIT, открытый код 👇
https://github.com/MrCipherSmith/helyx

## VC.ru — Пост

**Заголовок:** Helyx: управляй своим ИИ-ассистентом по коду через Telegram

**Текст:**

Если вы используете Claude Code для разработки, вы знаете проблему: инструмент работает только пока вы сидите у компьютера. Нет нормального способа проверить статус задачи, одобрить действие ИИ или отправить уточнение, не открывая терминал.

Helyx — это открытое решение этой проблемы. Self-hosted Telegram-бот, который соединяет Telegram с одной или несколькими сессиями Claude Code. Все взаимодействие — через чат: вы пишете инструкции с телефона, Claude работает с вашим кодом на сервере, результаты и статусы возвращаются в тот же тред.

**Для кого это:**

- Разработчики, которые хотят контролировать AI-сессии не находясь за компьютером
- Те, кто ведёт несколько проектов параллельно и хочет их разделить (форум-топики по проектам)
- Команды, которые self-hosted-ят AI-инструменты и хотят Telegram как интерфейс управления

**Что внутри:**

Двусторонний MCP-транспорт (stdio на хосте + HTTP в Docker), PostgreSQL как шина сообщений, pgvector для семантической памяти, веб-дашборд на React. Деструктивные действия Claude требуют явного подтверждения через инлайн-кнопки — это работает как «шлюз разрешений» для AI-агента.

Версия 1.47.0 добавляет маршрутизацию по форум-топикам и Skills Toolkit — механизм сохранения повторяемых AI-паттернов как переиспользуемых навыков.

Инструмент требует Docker, Bun и некоторого DevOps-опыта. Зато бесплатный, MIT, без внешних зависимостей кроме Telegram API и Anthropic API.

GitHub: https://github.com/MrCipherSmith/helyx
