# PRD: README Environment Variables and Prerequisites Fixes

## Overview

Fix two inaccuracies in the README: (1) the prerequisites table marks Ollama as required when it's optional, (2) the environment variables table is missing `CLAUDE_MODEL` and `MAX_TOKENS`.

## Problem Statement

**Issue 1 — Ollama marked as required:**
The prerequisites table shows Ollama as "Required: Yes". In practice, the bot runs without Ollama — it only affects semantic memory search (pgvector embeddings). Without Ollama, basic messaging, CLI sessions, standalone mode, and all non-memory features work fine. This causes users to think they need Ollama before they can start, creating unnecessary friction.

**Issue 2 — Missing env vars:**
The environment variables table is missing two vars that are generated by `claude-bot setup` and present in `.env.example`:
- `CLAUDE_MODEL` — the Claude model used for CLI sessions (default: `claude-sonnet-4-20250514`)
- `MAX_TOKENS` — max tokens per LLM response (default: `8192`)

**Issue 3 — OLLAMA_URL marked as required:**
The env vars table shows `OLLAMA_URL | Yes`, but the bot starts and functions without it.

## Goals

- Change Ollama prerequisite to "Optional (semantic memory search only)"
- Add `CLAUDE_MODEL` and `MAX_TOKENS` rows to the env vars table
- Change `OLLAMA_URL` from `Yes` to `No` in the env vars table
- Audit for any other missing or incorrect vars

## Non-Goals

- Changing any bot code
- Restructuring the README beyond these targeted fixes
